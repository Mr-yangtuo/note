# 静态笔记网站需求文档

## 一、项目目标

开发一个**移动端优先**的静态笔记网站。

主要用途：

- 在电脑端向项目目录添加笔记
- 部署后手机访问网址即可阅读
- 不需要在线编辑
- 不需要登录
- 不需要数据库
- 不需要后台管理

整个项目重点放在：

- 浏览体验
- 目录管理
- HTML内容渲染
- 后续易维护、易扩展

---

# 二、用户流程

电脑端：

新增课程/章节
    ↓
将HTML笔记放入 notes 目录
    ↓
重新部署
    ↓
手机访问网址
    ↓
浏览课程
    ↓
进入章节
    ↓
阅读HTML笔记

---

# 三、功能需求

## 1. 首页

展示所有课程。

例如：

生物化学
数字电子技术
大学物理
Python
数据库

点击进入课程。

---

## 2. 课程目录

展示课程目录。

例如：

生物化学

├── 第一章
│   ├── 第一节
│   ├── 第二节
│   └── 习题
│
├── 第二章
│   ├── 第一节
│   └── 第二节
│
└── 实验

要求：

- 支持无限层级
- 支持目录展开/折叠
- 当前所在章节高亮
- 点击最终节点进入笔记页面

---

## 3. 笔记页面

最终节点对应一个 HTML 文件。

例如：

第一节.html

网页需要直接渲染 HTML。

要求完整支持：

- 标题
- 图片
- 表格
- 数学公式
- 代码块
- CSS样式
- 锚点
- 超链接

保持 HTML 原有显示效果。

---

## 4. HTML渲染

HTML 文件全部由我自己生成。

因此：

- 不需要 Markdown 转换
- 不需要富文本编辑器
- 不需要内容解析

直接渲染 HTML。

---

## 5. 图片支持

HTML 中可能存在：

```html
< img src="./img/1.png">
```

要求：

- 图片正常显示
- 保持相对路径有效
- 图片宽度自适应手机屏幕

---

## 6. 数学公式

HTML 可能包含：

- MathJax
- KaTeX

要求正常显示。

---

## 7. 代码高亮

HTML 中可能包含：

```html
<pre><code class="language-python">
```

要求：

- 自动代码高亮
- 支持横向滚动
- 不影响手机阅读

推荐使用：

- highlight.js

---

## 8. 手机端体验

整个网站优先适配手机。

要求：

- Mobile First
- 字体舒适
- 点击区域足够大
- 阅读宽度合理
- 图片自适应
- 表格横向滚动
- 代码块横向滚动
- 长公式可滚动
- 页面滚动流畅

---

## 9. 搜索（预留）

后续可扩展搜索。

要求预留接口。

搜索范围：

- 课程名称
- 章节名称
- 文件标题

暂时不用搜索 HTML 正文。

---

# 四、目录组织方式

项目目录如下：

notes/

    生物化学/

        第一章/

            第一节.html

            第二节.html

            习题.html

        第二章/

            第一节.html

            第二节.html

    数字电子技术/

        第一章/

            门电路.html

要求：

- 文件夹代表目录
- html代表最终页面
- 支持无限层目录
- 后续新增课程无需修改代码

---

# 五、目录树生成

构建阶段自动扫描：

notes/

生成：

tree.json

例如：

[
  {
    "name":"生物化学",
    "children":[
      {
        "name":"第一章",
        "children":[
          {
            "name":"第一节",
            "file":"第一节.html"
          }
        ]
      }
    ]
  }
]

要求：

前端直接读取 tree.json。

不能手工维护目录。

---

# 六、路由设计

例如：

/

首页（课程列表）

/course/生物化学

课程目录

/note/生物化学/第一章/第一节

HTML笔记页面

要求：

- URL 可分享
- 可直接访问
- 支持浏览器刷新

---

# 七、技术方案

采用：

- React
- Vite
- TypeScript
- React Router
- Tailwind CSS

整个项目为静态网站。

不需要：

- Node后台
- Express
- Nest
- MySQL
- MongoDB

---

# 八、HTML加载方式

页面通过 fetch 动态读取：

notes/生物化学/第一章/第一节.html

然后直接渲染 HTML。

要求：

- 不解析 Markdown
- 不转换格式
- 保持 HTML 原样

---

# 九、UI设计要求

整体风格：

- 极简
- 清爽
- 阅读优先

参考：

- GitBook
- VitePress
- VuePress
- Obsidian Publish

要求：

- 白色背景
- 浅灰导航
- 蓝色链接
- 阅读区域居中
- 手机端优先
- 支持暗黑模式（预留）

---

# 十、项目结构

建议：

src/

    components/

    pages/

    hooks/

    utils/

    scripts/

notes/

要求：

- 组件职责单一
- TypeScript 严格模式
- 高内聚低耦合
- 易维护
- 易扩展

---

# 十一、不需要实现

以下功能全部不要：

- 登录
- 注册
- 用户系统
- 数据库
- Markdown编辑器
- 在线编辑
- 评论
- 收藏
- 点赞
- 后台管理
- 权限管理

这是一个纯静态笔记网站。

---

# 十二、扩展性要求

项目设计时应考虑未来扩展能力，包括但不限于：

- 全文搜索
- 收藏功能
- 标签系统
- 多主题切换
- PWA离线访问
- 多语言
- PDF导出
- 最近阅读
- 阅读进度记录

要求采用模块化设计，不影响现有架构。
